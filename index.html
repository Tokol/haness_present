The Bottleneck

Biomaterial Research papers contain formulation and property data, but turning hundreds of papers into ML-ready datasets is slow and difficult.

  • Data is scattered across text, tables, and supplementary files
  • Material names, units, and formulations are inconsistent
  • Manual extraction is repetitive and error-prone
  • Raw papers are not directly usable for ML training

Literature knowledge exists; structured CSV data is the bottleneck.

Agent Workflow

Each agent has a bounded job. The harness controls the decisions between them.

Paper Validation Agent

Checks biomaterial relevance, duplicates, safety text, and extraction readiness.

Decision: reject or continue

Source Extraction Agent

Reads PDF text, OCR pages, tables, and supplementary sources.

Decision: enough evidence?

Material/Formulation Extraction Agent

Extracts materials, roles, wt%, process settings, and measured properties.

Decision: evidence-backed row?

Training Builder

Filters measured-property rows and builds ML-ready CSV files.

Decision: exportable dataset?

Harness Assistant

Answers dataset questions and runs only explicit approved commands.

Decision: read-only or command?

Safety by Design

The model can suggest actions, but the harness decides what is allowed before anything touches files or data.

  • No arbitrary shell access
  • File writing stays inside project folders
  • Commands use an approved allowlist
  • Prompt-injection-like text is detected and ignored
  • API keys and secrets are not sent into prompts
  • Skills describe behavior; they are not executable code

Approved Commands

Natural-language prompts can request actions, but the harness only runs commands that are explicitly allowlisted.

Command Purpose
export_fortrain_zip Export ML training CSVs
export_dataset_zip Export structured datasets
export_audit_bundle_zip Export datasets and logs
validation_smoke_check Check dataset health

CLI or chat prompts are interpreted only into these approved commands. Anything outside the allowlist is refused or treated as a read-only question.

Extension Mechanism

The harness loads local skill specifications from the project.

skills/
  biomaterial_extraction/
    SKILL.md
  export_training_data/
    SKILL.md

biomaterial_extraction

Defines how papers, materials, formulations, and evidence should be handled.

export_training_data

Defines how clean CSVs and ML-ready export bundles should be prepared.

Skills are specifications, not executable plugins. The harness does not run untrusted skill code.
